# 条形码生成网站设计方案

## 1. 项目概述

- **项目名称**：条形码生成器
- **核心功能**：用户上传含编码和品名的Excel文件，生成可打印A4 PDF的条形码
- **目标用户**：企业内员工（进销存系统品项标签打印）
- **部署方式**：腾讯云轻量应用服务器（2核2G）+ Nginx

## 2. 技术方案

### 核心技术栈

| 技术 | 用途 |
|------|------|
| 原生 HTML + CSS + JavaScript | 前端页面 |
| JsBarcode | 生成 Code128 条形码 |
| jsPDF | 浏览器端生成 PDF |
| SheetJS (xlsx.full.min.js) | 解析 Excel 文件 |
| Nginx | 静态网站托管 |
| GitHub + Webhook | 代码管理 + 自动部署 |

### 关键决策

- **纯前端实现**：文件处理全在浏览器完成，服务器零负载
- **固定模板**：Excel 固定两列（编码、品名），无表头
- **无需登录**：任何人打开即可使用

## 3. 功能流程

## 4. Excel 模板格式

固定两列，无表头：
```
编码          品名
02060001     螺丝M6x30
02060002     螺母M8
```

**校验规则：**
- 编码列：纯数字，1-20位
- 品名列：非空，最大20字符
- 行数限制：最多500行/次

## 5. PDF 排版规格

| 参数 | 值 |
|------|-----|
| 纸张 | A4 (210mm × 297mm) |
| 边距 | 上下左右各 15mm |
| 品名字体 | 12pt，居中，位于条形码上方 |
| 编码字体 | 10pt，居中，位于条形码下方 |
| 条形码高度 | 20mm |
| 条形码宽度 | 按每行数量自适应 |
| 每页行数 | 4个≈13行 / 5个≈16行 / 6个≈19行 |

## 7. 部署架构

```
┌─────────┐    push    ┌──────────┐    webhook    ┌──────────────┐
│ GitHub  │ ─────────→ │ 腾讯云    │ ───────────→ │   Nginx      │
│ 仓库    │            │ 服务器    │              │ 静态托管      │
└─────────┘            └──────────┘              └──────────────┘
                                                       │
                                                       ↓
                                               用户访问网站
```

### 部署步骤
1. 服务器安装 Nginx
2. 域名解析到服务器IP（或直接用IP访问）
3. 申请免费SSL证书（Let's Encrypt）
4. 配置 Nginx HTTPS 反向代理
5. 设置 GitHub Webhook 自动拉取更新

## 8. 域名与SSL

| 项目 | 方案 | 费用 |
|------|------|------|
| 域名 | 腾讯云购买 .com 域名 | ~60元/年 |
| SSL证书 | Let's Encrypt 免费证书 | 免费 |
| 备选 | 直接用服务器IP访问 | 零费用 |
