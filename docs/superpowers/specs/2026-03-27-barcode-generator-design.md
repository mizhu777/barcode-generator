# 条形码生成网站设计方案
## 1. 项目概述

- **项目名称**：条形码生成器
- **核心功能**：用户上传包含编码和品名的 Excel / CSV 文件，生成可打印的 A4 PDF 条形码
- **目标用户**：企业内部需要批量打印物料条码的使用者
- **部署方式**：纯静态前端，可直接本地打开或部署到 Nginx 等静态站点

## 2. 技术方案
### 核心技术栈

| 技术 | 用途 |
|------|------|
| 原生 HTML + CSS + JavaScript | 页面与交互 |
| JsBarcode | 生成 Code128 条形码 |
| jsPDF | 生成 PDF |
| SheetJS | 解析 Excel / CSV 文件 |

### 关键决策

- **纯前端实现**：文件处理和 PDF 生成都在浏览器内完成
- **模板首行可带表头**：识别 `编码/品名` 等常见表头并自动跳过
- **高分辨率渲染**：页面按 300 DPI 生成位图后再嵌入 PDF，提高打印质量

## 3. 功能流程

1. 拖拽或点击上传 Excel / CSV 文件
2. 解析首个工作表并显示前 5 条预览数据
3. 识别并跳过首行表头
4. 选择每行条形码数量：4 / 5 / 6
5. 点击生成 PDF，浏览器自动下载

## 4. Excel 模板格式

固定前两列：

```text
编码,品名
02060001,螺丝M6x30
02060002,螺母M8
```

**校验规则：**
- 第一列：编码
- 第二列：品名
- 首行若为表头则自动跳过
- 空编码或空品名行会被忽略

## 5. PDF 排版规格

| 参数 | 值 |
|------|-----|
| 纸张 | A4 (210mm × 297mm) |
| 边距 | 上下左右各 15mm |
| 品名位置 | 条形码上方，居中 |
| 编码位置 | 条形码下方，居中 |
| 条形码高度 | 18mm |
| 页面渲染密度 | 300 DPI |
| 每页行数 | 约 8 行 |

## 6. 视觉设计

- 页面为单卡片结构，突出上传区和预览区
- 顶部使用渐变头图强化工具识别度
- 上传区、预览区和布局设置区采用一致的卡片化样式

## 8. 验收标准

- 上传带表头的 Excel / CSV 后，预览里不出现表头行
- 4/5/6 列布局可切换
- 生成的 PDF 内编码文字不重影
- 生成的 PDF 适合 A4 打印
- 页面支持拖拽上传
